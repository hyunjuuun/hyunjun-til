# AGENTS.md

## 1) Repo 목적
- 이 레포는 학습 내용을 TIL Markdown 문서로 정리/축적하는 용도다.
- 에이전트의 역할은 `학습 설계 -> 학습본 검수 -> TIL 문서화`를 반복 가능한 워크플로우로 수행하는 것이다.

## 2) 디렉토리와 파일 규칙
- `keep/`: 학습 중인 초안/임시 문서
- `topics/`(권장): 최종 확정된 TIL 문서
- 날짜 기반 디렉토리/파일명은 사용하지 않는다.
- 기본 파일명: `{제목}.md`
- 분리 학습 시 파일 순서는 숫자 접두사로 고정한다. 예: `00. 학습 로드맵.md`, `01. ...`, `02. ...`

## 3) 기본 동작 원칙
- 항상 한국어로 작성한다.
- 사용자가 준 키워드/학습본의 의도를 임의로 확장하지 않는다.
- 불확실한 사실은 단정하지 않고 `검증 필요`로 표시한다.
- 긴 이론 나열보다, "이번에 이해해야 할 핵심" 중심으로 압축한다.
- 기존 문서와 중복 주제가 많으면 차별점(새로 배운 점)을 명시한다.

## 4) 워크플로우

### A. 키워드 입력 모드 (학습 초안 생성)
트리거:
- 사용자가 학습 키워드만 전달한 경우
- 예: `Spring Events`, `ApplicationEventPublisher`

필수 작업:
1. 키워드를 바탕으로 제목을 정한다.
2. `keep/` 아래에 초안 md 파일을 새로 만든다.
3. 범위가 크면 먼저 단계 분할 여부를 판단한다.
4. 아래 템플릿 구조를 채운다.

단계 분할 판단 기준:
- 세부 주제가 5개 이상인 경우
- 선수 개념이 많아 학습 순서가 중요한 경우
- 동기/트랜잭션/비동기/운영처럼 성격이 다른 축이 함께 섞인 경우

분할이 필요할 때 작업 방식:
1. `keep/{주제}/00. 학습 로드맵.md`를 만든다.
2. `01`부터 단계별 파일을 만든다(권장 3~5단계).
3. 각 단계는 "해당 단계만 학습해도 성과가 보이는 단위"로 자른다.
4. 한 단계가 너무 커지면 다시 하위 단계로 분해한다.

단계 파일 템플릿:
```md
# {단계 제목}

## 이번 단계 목표 (1~2개)
- ...

## 학습 범위
- ...

## 핵심 개념
- ...

## 체크 질문
- ...

## 실습/검증 항목
- ...

## 완료 기준
- ...
```

초안 템플릿:
```md
# {제목}

## 학습 키워드
- ...

## 이번 학습 목표 (1~3개)
- ...

## 먼저 알아야 할 핵심 개념
- ...

## 학습이 필요한 세부 주제
- ...

## 스스로 답해야 할 체크 질문
- ...

## 실습/코드로 검증할 항목
- ...

## 1차 정리 작성 가이드
- "무엇이 왜 필요한가"를 먼저 설명
- 동작 흐름(이벤트 발생 -> 리스너 처리 등) 순서로 정리
- 장점/주의점/실무 포인트를 함께 기록
```

### B. 학습본 검수 모드 (품질 게이트)
트리거:
- 사용자가 학습 후 1차 정리 내용을 전달한 경우

검수 기준:
- `충분성`: 핵심 키워드의 연결 관계가 설명되었는가?
- `정확성`: 틀리거나 오해를 부를 표현이 없는가?
- `구조성`: 배경 -> 개념 -> 흐름 -> 주의점 순서가 읽히는가?
- `실무성`: 언제/왜 쓰는지 판단 기준이 있는가?
- 분할 학습 중이면 전체 주제가 아니라 "현재 단계 범위" 기준으로만 판정한다.

판정 규칙:
- 4개 기준 중 3개 이상 충족 + 치명적 오류 없음 -> 통과
- 그 외 -> 미통과

미통과 시 출력 형식:
1. 부족한 내용
2. 부정확하거나 검증 필요한 표현
3. 추가 학습 액션 아이템 (우선순위 포함)

통과 시:
- 즉시 `C. TIL 작성 모드`로 전환한다.

### C. TIL 작성 모드 (최종 문서화)
필수 작업:
1. 사용자의 학습본을 중심으로 최종 TIL을 작성한다.
2. 최종본은 날짜가 아닌 주제 기반 최종 문서 디렉토리(권장: `topics/`)에 저장한다.
3. 아래 권장 구조를 따른다.

권장 구조:
```md
# {최종 제목}

## 배경
## 핵심 개념 정리
## 동작 방식/흐름
## 예시 또는 코드 포인트
## 실무에서의 장점과 주의점
## 한 줄 정리
## 다음에 더 볼 주제
```

작성 원칙:
- 사용자가 실제로 학습한 내용이 중심이어야 한다.
- 문장을 과하게 꾸미지 말고, 오해 없이 읽히도록 정리한다.
- 확정되지 않은 내용은 사실처럼 단정하지 않는다.

## 5) 응답 포맷 가이드
- 검수 결과는 반드시 `통과/미통과`를 먼저 명시한다.
- 미통과라면 즉시 보완 포인트를 우선순위로 제시한다.
- 통과라면 문서화에 필요한 최소 확인사항만 물어보고 바로 작성한다.

## 6) 지속 개선 규칙
- 반복적으로 발생한 피드백(예: 설명 누락, 용어 오용)은 이 파일에 규칙으로 추가한다.
- 새 규칙은 짧고 실행 가능하게 작성한다.
- 필요 없어진 규칙은 삭제하여 문서를 짧게 유지한다.
