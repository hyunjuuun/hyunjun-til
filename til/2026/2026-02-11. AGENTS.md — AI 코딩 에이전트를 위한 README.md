# AGENTS.md — AI 코딩 에이전트를 위한 README

## Why — 왜 AGENTS.md가 필요한가?

AI 코딩 에이전트는 강력하지만 다음과 같은 한계를 가진다.

- 세션 간 컨텍스트를 유지하지 않는다.
- 프로젝트 구조를 자동으로 이해하지 못한다.
- 우리 팀의 코드 컨벤션이나 워크플로우를 모른다.
- 매번 프롬프트로 모든 규칙을 설명해야 한다.

즉, 에이전트는 “코드를 잘 쓰는 개발자”지만
“우리 프로젝트를 아는 개발자”는 아니다.

이 문제를 해결하기 위해 등장한 것이 **AGENTS.md**이다.

- README.md → 인간 개발자를 위한 문서
- AGENTS.md → AI 에이전트를 위한 운영 지침서

참고 문서

- https://agents.md/
- https://developers.openai.com/codex/guides/agents-md/

---

## What — AGENTS.md란 무엇인가?

AGENTS.md는 프로젝트 루트 또는 서브 디렉토리에 위치하며,

코딩 에이전트가 자동으로 탐색하여 읽는 파일이다.

이 파일은 에이전트에게 다음을 알려준다:

- 프로젝트 개요
- 아키텍처 구조
- 빌드 및 테스트 방법
- 작업 워크플로우
- 품질 게이트
- 참조해야 할 다른 문서들

즉,

“이 프로젝트에서 일할 때 반드시 따라야 할 규칙”을 정의하는 파일이다.

---

## 설정 방식

### 1. 기본 생성 방식

Codex 기준으로 `/init` 명령어를 실행하면 현재 작업 디렉토리에 AGENTS.md가 생성된다.

### 2. 글로벌 설정

`~/.codex/AGENTS.md` 위치에 파일을 두면 모든 프로젝트에 공통 규칙이 적용된다.

---

## AGENTS.md에 무엇을 넣어야 할까?

### 포함하면 좋은 것

- 프로젝트 개요 및 구조
- 빌드/테스트 명령어
- 코드 스타일 규칙
- 금지 사항 (예: 직접 DB 접근 금지)
- 사용 가능한 CLI tools
- MCP 서버 정보
- 구현 워크플로우 정의
- Quality Gate

## Best Practices

### 1. Keep it brief and focused

- 100라인 이하 권장
- 길수록 오히려 혼란을 유발할 수 있음

---

### 2. Continuously Update with Real Mistakes

에이전트가 반복적으로 실수하는 패턴이 있다면

- 프롬프트에서 계속 고치지 말고
- AGENTS.md에 룰로 추가하라

AGENTS.md는 일회성 파일이 아니라 지속적으로 발전하는 운영 문서다.

---

### 3. Task-Specific 문서와 연결하라

AGENTS.md는 범용 규칙만 담는다.

구체적인 작업은 별도의 md 파일로 관리한다.

예:

- Plans.md
- Architecture.md
- API_Spec.md

AGENTS.md 내부에서는 다음처럼 연결할 수 있다.

```
If implementing API, follow:
- docs/API_Spec.md
```
