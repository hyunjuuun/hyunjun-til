# 학습 에이전트 모드

## 목적
- `keep/` 하위의 단계별 초안 문서를 실제 학습 세션으로 전환한다.
- 학습 에이전트(AI 어시스턴트)를 "설명 + 질문 + 검수 + 다음 액션 제안"을 수행하는 역할로 사용한다.
- 특정 주제에 종속되지 않는 공용 워크플로우를 제공한다.

## 언제 쓰는가
- 키워드 기반 초안(`00`, `01`, `02`...)이 이미 생성된 상태
- 지금부터 한 단계씩 실제 학습을 진행하려는 상태

## 모드 진입 프롬프트
아래 템플릿으로 시작한다.

```text
[학습 에이전트 모드 시작]
대상 파일: {keep 하위 단계 파일 경로}
```

예시:
```text
[학습 에이전트 모드 시작]
대상 파일: keep/스프링 이벤트 처리(Spring Events)/01. 이벤트 모델과 기본 발행-구독.md
```

## 에이전트 응답 형식(고정)
학습 에이전트는 매 세션에서 아래 순서로 답한다.

1. 오늘의 학습 범위(과다 확장 금지, 대상 파일 범위만)
2. 핵심 개념 3개(짧게)
3. 이해 점검 질문 3개(즉답형 + 설명형 혼합)
4. 미니 실습 1개(또는 코드 독해 1개)
5. 통과 기준(이번 단계 완료 조건 재확인)

## 세션 진행 루프
1. `설명 요청`: 개념을 짧게 설명받는다.
2. `내가 답변`: 체크 질문에 스스로 답한다.
3. `검수 요청`: 에이전트가 정확성/충분성 피드백을 준다.
4. `보완 학습`: 부족한 포인트만 추가 학습한다.
5. `통과 판정`: 통과 시 다음 단계로 이동한다.

## 세션 제어 프롬프트(권장)
- `핵심만 설명해줘.`
- `예시 코드 포함해서 설명해줘.`
- `내 답변 검수해줘.`
- `미니 퀴즈 3개 내줘.`
- `지금 단계 기준으로 통과/미통과 판정해줘.`
- `미통과면 보완 학습 항목 3개만 우선순위로 줘.`

## 단계 종료 체크리스트
- [ ] 단계 문서의 `완료 기준`을 충족했다.
- [ ] 핵심 개념을 내 언어로 3~5문장 설명할 수 있다.
- [ ] 체크 질문에 막힘 없이 답할 수 있다.
- [ ] 실습/검증 항목 1개 이상 수행했다.

## 세션 기록 템플릿
이 템플릿은 학습자가 세션 내용을 구조적으로 기록하기 위한 양식이다.
에이전트는 사용자가 공유한 기록을 바탕으로 검수/보완 피드백을 제공한다.

```md
### Session {번호}
#### 대상 단계
- {파일 경로}

#### 에이전트 답변 핵심 요약
- ...

#### 내가 이해한 내용(내 언어)
- ...

#### 헷갈리는 부분
- ...

#### 에이전트 검수 결과
- 통과/미통과:
- 사유:

#### 다음 액션
- ...
```

## 최종 TIL 전환 조건
- 각 단계가 모두 `통과` 상태여야 한다.
- 통과한 단계 요약(핵심/주의점/예시)을 모아 최종 TIL 작성 모드로 전환한다.
